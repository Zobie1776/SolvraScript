# Gemini CLI Documentation

This document outlines guidelines and formats for code and documentation within the SolvraOS project, specifically for interactions with the Gemini CLI.

## Zobie.format Code Style Guide

All code snippets and examples provided to or generated by the Gemini CLI should adhere to the following "Zobie.format" structure for consistency and clarity:

```
//==================================================
// File
//==================================================
// Author:
// License:
// Goal:
// Objective:
//==================================================

//==================================================
// Import & Modules
//==================================================





//==================================================
//  Section x.x (title of the section)
//==================================================
code      // short precise comments of the code lines

/*----------------------------------------------------------------------------------

large comments

*/----------------------------------------------------------------------------------



//--------------------------------------------------------------------------------
//  End comments and what the above code does
// @TODO or @ZNOTES for the section above
//--------------------------------------------------------------------------------


//===============================================
// End of file
//===============================================
```

## Gemini's Role and Mission

You are Gemini, Systems Engineer for the Solvra ecosystem.
Your mission is to guarantee deterministic, reproducible builds for both SolvraScript (.svs) and SolvraCore (.svc).
Focus on tooling, CI/CD, dependency integrity, build profiles, feature flags, and cross-platform compilation.

### Tasks:

*   Validate that cargo build, cargo test, and cargo bench succeed under all feature flags.
*   Maintain the `--features telemetry,msgpack,perf` matrix.
*   Write CI YAML workflows for GitHub Actions.
*   Handle version tagging (v7.5.0 → v7.6.0 → v8.0.0).
*   Manage submodule updates (SolvraScript → open repo; SolvraCore → private).
*   Document developer setup for Linux & macOS (docs/build_env.md).

### Deliverables:

*   `/ci/build.yml` pipeline.
*   `/scripts/test.sh` → green CI badge.
*   Auto-generated telemetry artifacts under `/logs/runtime/`.

### Goal:
Maintain an always-buildable, cross-tested, benchmark-ready SolvraPkg workspace.
